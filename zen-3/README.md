# Zen — Benefits Navigator (Local MVP)

**Find your calm. Find your way.** From crisis to confirmed help — AI that closes the loop.

USAII Global AI Hackathon 2026 · Undergraduate · Brief 4 (Public Services) · Direction A (Benefits Navigator)
Team: **Steff** (Data Science + Math) · **Diego** (Software Engineering)

---

## What this is

A **fully functional local MVP**. It runs end-to-end on your laptop:

voice/text intake → NLP extraction + confidence gate → **MILP fair allocation**
→ ranked plan with regulatory citations → 24/72h loop check-in → escalation →
ONG outcome dashboard.

The hard part (the optimization engine with the demographic-parity constraint)
is **real and runs**. This is the base you two modify: Steff tunes the IDM /
data-science models, Diego owns hosting, the real DB, and production infra.

## Run it locally (2 minutes)

```bash
# 1. install
pip install -r requirements.txt

# 2. generate resource data (seed = no internet needed)
python scraper.py --seed

# 3. start the server
uvicorn app:app --reload

# 4. open the app
#    http://127.0.0.1:8000
```

That's it. The web app talks to the FastAPI backend on the same origin.

## Two worlds, one system (how it maps to the brief)

The app has a mode toggle at the top:

- **"I need help" (the protagonist)** — the brief's core: a person describes
  their situation, confidence-gated intake, fair MILP match, plain-language
  eligibility citations, loop closure. This is the heart of the demo.
- **"I'm an organization" (support)** — ONGs register/update real capacity
  (so matching uses live spots, not dead data) and see a demand forecast by
  zone + service to pre-position supply. Answers "where does capacity come
  from?" without becoming a second product.

The person flow is the star; the ONG view is deliberately secondary.

## The data flow (what to show a judge)

1. **Intake** — Web Speech API captures voice client-side (no key, works in
   Chrome) → transcript posted to `/api/intake`. The audio is never sent or
   stored; only the text. The NLP layer returns needs, urgency, language, a
   **confidence score**, and a **safety flag**.
2. **Confidence gate** — ≥80% proceeds; 50–80% asks to confirm; <50% asks to
   redo (human after 2 tries); safety keywords → straight to a human.
3. **Match** — `/api/match` runs the **MILP**: maximize urgency-weighted
   coverage s.t. capacity, eligibility, distance, and **demographic parity**.
4. **Plan** — ranked programs with address, hours, confidence, and the exact
   regulatory citation for *why you qualify*. Stale records (>45d) are flagged.
5. **Demo** — `/api/demo` returns the naive-vs-fair scarcity comparison
   (gap 20% → 10%).
6. **Check-in / escalation** — `/api/checkin`; a "no" triggers caseworker
   reassignment (the loop closure).
7. **Dashboard** — `/api/dashboard` returns outcome metrics + the equity audit.

### Diego (Software Engineering)

- **DB**: replace `data/resources.json` with a real store. Only `load_resources()`

- **Voice in production**: swap the browser Web Speech API for Whisper behind a
  `/api/transcribe` endpoint (audio in, text out, audio discarded).

## Responsible AI (built in, defend it)
- Audio is **never stored** — only the transcript text is processed.
- The system **never auto-rejects** and **never profiles** (EU AI Act Annex III
  → designed as a *preparatory* tool, not a decision-maker).
- **Demographic-parity** constraint in the MILP — the safeguard the VI-SPDAT
  never had (it was retired by its creators for racial bias).
- Three human-in-the-loop triggers: safety keywords, persistent low confidence,
  broken loop.

## Note on the scraper
This sandbox built the seed data; run `python scraper.py --live --url <...>`
from a machine with internet to pull real listings. Only scrape sources whose
terms permit it; prefer official open-data / HSDA feeds.

## Latest additions (post-retro corrections)

- **Plain language + readability metric** (`engine/text_tools.py`): every "why you
  qualify" is shown in plain words behind a "?" icon, with a Flesch reading-ease
  score proving it (≈6th–7th grade vs. college-level legal text).
- **Relevant follow-up questions** (`engine/followups.py`): the brief requires
  "ask relevant questions"; after intake we ask 2–3 plain-language eligibility
  questions and interpret them into live eligibility signals.
- **Two-tier routing, visible**: emergencies → real-time heuristic; scarce
  resources → MILP nightly batch. The route is shown to the user.
- **Visible MILP batch** (`/api/batch/run`): the Organization view shows the solver
  working — each person as a square, served/faded, naive vs. fair side by side.
- **Caseworker view** (`engine/caseworker.py`): vulnerability-triaged escalation
  queue (not FIFO), with call + resolve. A broken loop creates a real escalation.
- **Case number + optional contact**: each plan gets a ZEN-2026-XXXX number; email
  and partner-ONG contact are both **opt-in and optional**. Recover a plan later by
  entering the case number (`/api/case/...`).
- **Gemini chat with fallback** (`/api/chat`): set your key in `gemini_key.txt`
  (git-ignored) or the `GEMINI_API_KEY` env var. No key / no internet → automatic
  fall back to transparent rule-based answers, so the demo never breaks. The key is
  NEVER exposed to the frontend.
- **Deploy**: `render.yaml` included — connect the repo on Render, set
  `GEMINI_API_KEY` in the dashboard, done.
